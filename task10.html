@supports (display: grid) { .container { display: grid; } }
This checks if the browser supports CSS Grid before applying the layout.

Questions & Answers:

1. Why do we check browser support?
To avoid breaking layouts in older browsers that don't support new features.
2. What happens if Grid is not supported?
The fallback layout (usually block or flex) is used instead.
3. Is this needed for all projects?
No, mostly for projects that support many browsers or older devices.
4. Who uses this in real projects?
Companies with large user bases: e-commerce, banking, government, enterprise.
5. What does safe CSS mean?
Writing CSS that works everywhere or has fallback behavior to prevent broken UI.
